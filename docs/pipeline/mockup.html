User Stories

Dashboard
US-1.1
Vue d'ensemble avec statistiques globales
En tant que chercheur, je peux voir les statistiques globales (nombre de chercheurs, themes, clusters) sur le dashboard principal pour avoir un apercu rapide de la communaute.

Criteres d'acceptation :
• Le dashboard affiche 4 cartes: Chercheurs, Themes, Clusters, Publications
• Les chiffres se mettent a jour en temps reel
• Chaque carte est cliquable et mene a la section correspondante
US-1.2
Activite recente visible
En tant que chercheur, je peux voir l'activite recente (nouveaux profils, dernieres publications) pour rester informe.

Criteres d'acceptation :
• La liste affiche les 5 dernieres activites avec avatar, nom, action, timestamp
• Les activites sont triees par date decroissante
• Cliquer sur un nom ouvre le profil du chercheur
US-1.3
Mini-carte interactive sur le dashboard
En tant que chercheur, je peux cliquer sur l'apercu de la carte pour naviguer vers la carte thematique complete.

Criteres d'acceptation :
• L'apercu affiche un apercu simplifie des clusters
• Au survol, l'apercu est mis en evidence avec un curseur pointer
• Au clic, navigation vers l'ecran Carte Thematique
US-1.4
Statistiques detaillees
En tant que chercheur, je peux voir des statistiques detaillees (distribution des themes, metriques de similarite, tendances temporelles).

Criteres d'acceptation :
• Graphique en barres pour la distribution des themes
• Graphique en ligne pour les tendances temporelles
• Histogramme des scores de similarite
• Les graphiques sont interactifs (survol avec tooltip)
Chercheurs
US-2.1
Recherche et filtres sur la liste
En tant que chercheur, je peux rechercher et filtrer la liste des chercheurs par laboratoire, theme ou mot-cle pour trouver des collegues.

Criteres d'acceptation :
• Champ de recherche textuelle avec resultats en temps reel
• Filtres par laboratoire (dropdown) et par theme (dropdown)
• Les filtres se combinent (ET logique)
• Nombre de resultats affiche
US-2.2
Profil complet du chercheur
En tant que chercheur, je peux voir un profil complet (infos, themes, publications, position sur la carte) pour comprendre les travaux d'une personne.

Criteres d'acceptation :
• Sidebar gauche avec avatar, nom, labo, bio
• Zone droite avec mots-cles en tags et liste de publications
• Bouton "Voir sur la carte" mene a la carte
• Boutons Modifier et Comparer disponibles
US-2.3
Formulaire d'ajout/modification
En tant que chercheur, je peux ajouter ou modifier mon profil via un formulaire structure (nom, labo, mots-cles, bio, publications).

Criteres d'acceptation :
• Champs: Nom, Laboratoire (select), Mots-cles (tag input), Bio (textarea)
• Publications en blocs repetables (titre, co-auteurs, revue/conference)
• Validation des champs obligatoires
• Boutons Enregistrer et Annuler
• Bandeau jaune indiquant que le profil sera soumis a validation admin
• Le profil n'est publie qu'apres approbation par un administrateur
US-2.4
Comparaison cote-a-cote
En tant que chercheur, je peux comparer deux chercheurs cote a cote avec les themes communs et un score de similarite.

Criteres d'acceptation :
• Deux profils en colonnes avec un gauge de similarite au centre
• Themes communs mis en evidence visuellement (contour)
• Score affiche en pourcentage dans un cercle
• Section recapitulative des themes partages
US-2.5
Navigation profil vers carte
En tant que chercheur, je peux naviguer du profil vers la position de la personne sur la carte thematique.

Criteres d'acceptation :
• Bouton "Voir sur la carte" dans le profil
• La carte s'ouvre centree sur le chercheur selectionne
• Le point du chercheur est mis en evidence sur la carte
Carte Thematique
US-3.1
Carte interactive avec zoom et filtres
En tant que chercheur, je peux voir la carte des clusters interactifs avec zoom, pan et controles de filtrage.

Criteres d'acceptation :
• Carte SVG avec clusters colores et points chercheurs
• Panneau de filtres flottant (theme, labo)
• Legende des couleurs
• Zoom et pan fonctionnels
US-3.2
Clic sur cluster pour voir les membres
En tant que chercheur, je peux cliquer sur un cluster pour voir ses chercheurs et themes.

Criteres d'acceptation :
• Au clic sur un cluster, une infobulle liste les membres
• Les themes du cluster sont affiches
• Chaque nom de chercheur est un lien vers son profil
US-3.3
Clic chercheur vers profil
En tant que chercheur, je peux cliquer sur un point de chercheur sur la carte pour naviguer vers son profil.

Criteres d'acceptation :
• Chaque point de chercheur est cliquable
• Au survol, le nom du chercheur s'affiche
• Au clic, navigation vers la page de profil
US-3.4
Explorer les themes en liste
En tant que chercheur, je peux parcourir les themes en vue liste avec clusters, nombre de chercheurs et liens croises.

Criteres d'acceptation :
• Grille de cartes par cluster avec nom, comptage, sous-themes
• Cartes extensibles au clic pour voir les membres
• Liens bidirectionnels vers les profils et la liste chercheurs
• Code couleur coherent avec la carte
Connexion & Profil
US-5.1
Connexion utilisateur
En tant qu'utilisateur, je peux me connecter pour acceder a mes fonctionnalites personnelles (edition de profil pour les chercheurs, panneau d'administration pour les admins).

Criteres d'acceptation :
• Formulaire de connexion centre (email + mot de passe)
• Boutons de demo pour simuler la connexion en tant que chercheur ou admin
• Apres connexion, le bouton "Connexion" est remplace par le nom de l'utilisateur avec avatar
• Un onglet "Admin" apparait dans la navbar pour les administrateurs uniquement
• Menu deroulant avec "Mon profil" et "Deconnexion"
US-5.2
Edition de profil par le chercheur
En tant que chercheur connecte, je peux modifier mon propre profil et mes modifications seront validees par un administrateur.

Criteres d'acceptation :
• Le bouton "Modifier" est actif sur mon propre profil
• Le bouton "Modifier" est verrouille sur les profils d'autres chercheurs
• Le bouton "Modifier" est cache si non connecte
• Le formulaire d'ajout affiche un bandeau d'avertissement jaune
• Le profil soumis apparait dans "Profils en attente" cote admin
Administration
US-4.1
Gestion des utilisateurs et roles
En tant qu'admin connecte, je peux gerer les utilisateurs (attribuer les roles, approuver les profils en attente).

Criteres d'acceptation :
• Accessible uniquement apres connexion
• Tableau avec nom, email, role (badge), statut
• Onglet "Profils en attente" avec boutons Approuver/Rejeter
• Roles modifiables via dropdown
• Possibilite de revoquer un acces
US-4.2
Import en masse (CSV, Google Scholar)
En tant qu'admin connecte, je peux importer des chercheurs en masse via upload CSV/Excel ou integration Google Scholar.

Criteres d'acceptation :
• Zone de depot drag & drop pour fichiers CSV/Excel
• Champ URL Google Scholar avec bouton Importer
• Tableau de previsualisation des donnees importees
• Bouton de confirmation avec comptage
• Lien vers les logs apres import
US-4.3
Configuration (langue, seuils, NLP)
En tant qu'admin connecte, je peux configurer les parametres de l'app (langue FR/EN, seuils de similarite, choix d'algorithme NLP).

Criteres d'acceptation :
• Section Langue: boutons radio FR/EN
• Section Seuil: slider avec valeur affichee (0.0 - 1.0)
• Section NLP: dropdown (TF-IDF, Word2Vec, BERT)
• Bouton Sauvegarder
US-4.4
Journaux d'audit
En tant qu'admin connecte, je peux voir les logs d'audit indiquant qui a modifie quoi et quand.

Criteres d'acceptation :
• Tableau avec colonnes: Date/Heure, Utilisateur, Action, Detail
• Filtres par plage de dates
• Actions codees par couleur (Ajout, Modification, Suppression, Import)
• Pagination pour les longs historiques
